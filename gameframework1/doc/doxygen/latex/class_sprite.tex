\doxysubsection{Detailed Description}
Classe qui représente un élément d\textquotesingle{}animation graphique 2D. 

Cette classe met à disposition différentes méthodes permettant de gérer simplement un sprite animé. Un sprite est un élément graphique qui affiche une image bitmap. Cet élément graphique doit appartenir à une scène. Il possède un certain nombre de propriétés, associées à des méthodes, influençant son apparence et sa position \+:
\begin{DoxyItemize}
\item {\ttfamily pos} \+: la position 2D du sprite
\item {\ttfamily x} \+: la position sur l\textquotesingle{}axe horizontal du sprite
\item {\ttfamily y} \+: la position sur l\textquotesingle{}axe vertical du sprite
\item {\ttfamily z} \+: l\textquotesingle{}ordre z du sprite
\item {\ttfamily rotation} \+: l\textquotesingle{}angle de rotation du sprite
\item {\ttfamily scale} \+: un facteur de grossissement/rappetissment du sprite
\item {\ttfamily opacity} \+: l\textquotesingle{}opacité du sprite, valeur décimal comprise en 0 et 1, 0 étant transparent et 1 opaque.
\end{DoxyItemize}

Toutes ces propriétés peuvent être animées de façon très simple au moyen d\textquotesingle{}un objet QProperty\+Animation.

D\textquotesingle{}autres méthodes permettent de connaître la géométrie et le placement du sprite \+: width(), height(), pos\+X(), pos\+Y(), left(), right(), top(), bottom(). Il est possible d\textquotesingle{}obtenir le rectangle dans lequel le sprite est inscrit avec global\+Bounding\+Box(). Il est possible d\textquotesingle{}obtenir la forme exacte du sprite avec global\+Shape().

L\textquotesingle{}apparence du sprite n\textquotesingle{}est déterminée que par une seule image. Toutefois, il est possible d\textquotesingle{}en mémoriser plusieurs, afin de changer facilement d\textquotesingle{}apparence. Il est également possible de faire changer automatiquement ces images dans le but d\textquotesingle{}obtenir un sprite animé.

La méthode \mbox{\hyperlink{class_sprite_ad382222eee275aa4bf98bdbeb6b0fc66}{add\+Animation\+Frame()}} permet d\textquotesingle{}ajouter une image au sprite. Si plusieurs images sont ajoutées, elles sont conservées dans une liste qui préserve l\textquotesingle{}ordre d\textquotesingle{}ajout des images.

La méthode \mbox{\hyperlink{class_sprite_a7dc5279506894215965f386b01e4dd2b}{set\+Current\+Animation\+Frame()}} permet de spécifier quelle image doit être affichée (l\textquotesingle{}indice de la première image est 0). La méthode current\+Animation\+Frame() indique quelle image est actuellement affichée par le sprite.

La méthode \mbox{\hyperlink{class_sprite_ac7d053af8f5cf1221d3cf016a4c65874}{start\+Animation()}} permet de démarrer l\textquotesingle{}animation des images. La méthode \mbox{\hyperlink{class_sprite_a7fc484c57aa07b95311eacf27d789a0c}{stop\+Animation()}} permet de stopper l\textquotesingle{}animation des images. La vitesse d\textquotesingle{}animation peut être réglée avec \mbox{\hyperlink{class_sprite_aa27e28e0a1ed0153f4fcc9a9a3402e0f}{set\+Animation\+Speed()}} ou au moment de démarrer l\textquotesingle{}animation.

Il est également possible de demander au sprite d\textquotesingle{}émettre un signal chaque fois que l\textquotesingle{}animation est terminée, avec la méthode \mbox{\hyperlink{class_sprite_a93d7b808a1bcb070fa0b484a9f0ab35b}{set\+Emit\+Signal\+End\+Of\+Animation\+Enabled()}}. Cela permet par exemple de connecter ce signal au slot delete\+Later() du même objet, afin de détruire automatiquement le sprite dès que l\textquotesingle{}animation est terminée (par exemple pour afficher une explosion).

Comme la classe \mbox{\hyperlink{class_sprite}{Sprite}} spécialise la classe QGraphics\+Pixmap\+Item, qui elle-\/même spécialise la class QGraphics\+Item, toutes les méthodes de QGraphics\+Item sont accessibles au sprite.\hypertarget{class_sprite_sprite_on_scene}{}\doxysubsection{Intéger un sprite à une scène}\label{class_sprite_sprite_on_scene}
Lorsqu\textquotesingle{}un sprite est créé, il faut le placer sur une scène pour qu\textquotesingle{}il apparaisse, au moyen de la méthode \mbox{\hyperlink{class_game_scene_a0a63f65536de7e4d07be813b48f8997c}{Game\+Scene\+::add\+Sprite\+To\+Scene()}}. Celle-\/ci en prend alors la propriété et se chargera de la détruire lorsque la scène sera détruite.

Une fois ajouté à la scène, il est possible de l\textquotesingle{}abonner au tick afin qu\textquotesingle{}il reçoive la cadence du jeu, grâce à la méthode \mbox{\hyperlink{class_game_scene_a0bfc0ba479a5b0388a737480e8871cbe}{Game\+Scene\+::register\+Sprite\+For\+Tick()}}. Dès qu\textquotesingle{}un sprite est abonné à la cadence, sa méthode virtuelle \mbox{\hyperlink{class_sprite_a613ffe7e1cfea09977d5985104a94d46}{tick()}} est automatiquement appelée.

Attention \+: si le sprite est détruit, il faut s\textquotesingle{}assurer au préalable de le désabonner avec \mbox{\hyperlink{class_game_scene_aa40d3ff9e3b7c8296d91ef98d98358d1}{Game\+Scene\+::unregister\+Sprite\+From\+Tick()}}.\hypertarget{class_sprite_tick_handler}{}\doxysubsection{Le gestionnaire de cadence}\label{class_sprite_tick_handler}
Un sprite peut être déplacé de plusieurs façons différents au sein d\textquotesingle{}une scène.

Une première solution est qu\textquotesingle{}un élément externe se charge de la position d\textquotesingle{}un sprite en utilisant les propriétés publiques du sprite, telle que x (set\+X()), y (set\+Y()) et z (set\+Z()) ou la fonction set\+Pos().

Une autre solution est de laisser le sprite gérer seul son déplacement en se basant sur la cadence du jeu.

Pour faire cela, il suffit d\textquotesingle{}attribuer au sprite un gestionnaire de cadence (\mbox{\hyperlink{class_sprite_tick_handler}{Sprite\+Tick\+Handler()}}) qui contiendra le code nécessaire pour faire bouger le sprite.

Si un gestionnaire de cadence est attribué au sprite, à chaque tick, le gestionnaire de cadence est appelé (\mbox{\hyperlink{class_sprite_a613ffe7e1cfea09977d5985104a94d46}{tick()}}).

Une dernière solution est de spécialiser la classe \mbox{\hyperlink{class_sprite}{Sprite}} afin de surcharger la méthode \mbox{\hyperlink{class_sprite_a613ffe7e1cfea09977d5985104a94d46}{tick()}}.
