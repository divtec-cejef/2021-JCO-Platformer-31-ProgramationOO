◆ resourcesPath()

QString GameFramework::resourcesPath ( )

Cette fonction retourne le chemin absolu du répertoire res. Si la pseudo-constante DEPLOY est définie, elle se base sur la structure de répertoires suivante : 

+--ce_projet/
   |
   +--res/
   +ce_projet.exe

Si la pseudo-constante DEPLOY n'est pas définie, elle se base sur la structure de répertoires suivante sous Windows : 

+--pendu/
   |
   +--build-ce_projet.../
   |  |
   |  +--debug/
   |  |  |
   |  |  +ce_projet.exe
   |  |
   |  +--release/
   |     |
   |     +ce_projet.exe
   |
   +--doc/
   +--res/
   +--src/

Si la pseudo-constante DEPLOY n'est pas définie, elle se base sur la structure de répertoires suivante sous Linux : 

+--pendu/
   |
   +--build-ce_projet.../
   |  |
   |  +ce_projet.exe
   |
   +--doc/
   +--res/
   +--src/

La pseudo-constante DEPLOY ne devrait être définie que lors de la génération de l'exécutable en vue du déploiement.

Cette constante peut être définie au moment de la génération des MakeFile, avec la commande qmake. Voici un exemple d'utilisation possible : 

qmake -config release "DEFINES+=DEPLOY" ..\..\src\GameFramwork.pro

Cette commande génère les fichiers MakeFile, nécessaires pour l'étape de compilation, en s'assurant que la compilation se fasse en mode Release et que le pseudo-constante DEPLOY soit définie.

Renvoie
une chaîne de caractères contenant le chemin absolu du répertoire res.
